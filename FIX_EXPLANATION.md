# T-Lumina Binary Load ERROR - Root Cause & Fix

## 🔴 The Error
```
Loading T-Lumina Model...
[CRITICAL ERROR] Missing Layer in binary file: model.norm.weight
```

---

## 🔍 Root Cause Analysis

### The Problem
There was a **data type mismatch** between the export script and the model loader:

1. **Export Script** (`export_direct.py`) saves weights as **FP16** (half precision, 2 bytes per float):
   ```python
   fp16_tensor = param.to(torch.float16)  # ← Converts to FP16
   f.write(struct.pack("I", 1))           # ← Type 1 = FP16
   ```

2. **Loader** (`core/model.cpp`) incorrectly assumed **FP32** (4 bytes per float):
   ```cpp
   t.size = raw.data.size() / 4;  // ❌ WRONG! Assumes 4 bytes (FP32)
   ```

### Why This Causes "Missing Layer"
- When dividing FP16 data (2 bytes/element) by 4, the size calculation becomes **half of what it should be**
- This caused the tensor to be interpreted as corrupted or missing
- The error message was misleading - the layer existed but was parsed incorrectly

---

## ✅ The Fix

### Added FP16→FP32 Conversion Function
```cpp
inline float fp16_to_fp32(uint16_t h) {
    // Converts IEEE 754 half-precision to single-precision
    uint32_t sign = (h & 0x8000) << 16;
    uint32_t exp  = (h & 0x7C00) >> 10;
    uint32_t mant = (h & 0x03FF);
    
    // ... (handles denormalized numbers, infinity, etc.)
    
    float result;
    std::memcpy(&result, &f, sizeof(result));
    return result;
}
```

### Updated extract_fp32() Function
```cpp
void extract_fp32(Tensor& t, const std::string& name, ...) {
    const TensorRaw& raw = raw_tensors[name];
    
    if (raw.type == 1) {  // Type 1 = FP16
        t.size = raw.data.size() / 2;  // ✅ 2 bytes per element
        t.data = new float[t.size]();
        const uint16_t* fp16_data = reinterpret_cast<const uint16_t*>(raw.data.data());
        for (int i = 0; i < t.size; ++i) {
            t.data[i] = fp16_to_fp32(fp16_data[i]);  // ✅ Convert to FP32
        }
    } else {  // Assume FP32
        t.size = raw.data.size() / 4;
        t.data = new float[t.size]();
        std::memcpy(t.data, raw.data.data(), raw.data.size());
    }
}
```

## 📌 Key Learnings

| Aspect | FP16 | FP32 |
|--------|------|------|
| **Bytes per element** | 2 | 4 |
| **Precision** | Half | Full |
| **Export script** | Uses `torch.float16` | N/A |
| **Loader division** | `size / 2` | `size / 4` |

---

## 🚀 Prevention Tips

1. **Always check data types** when exporting and loading ML models
2. **Add type detection** in loader (don't hardcode assumptions)
3. **Validate dimensions**: 
   - Total bytes ÷ bytes-per-element should match expected tensor size
   - `print(raw_tensors[name].data.size())` when debugging

4. **Consistent export/load chain**:
   ```
   PyTorch Model → Export (FP16) → Binary File → Load (FP16→FP32) → C++ Model
   ```
